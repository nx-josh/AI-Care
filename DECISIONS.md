### 🟡 UX-2 응답 스트리밍 vs 일괄

### 🟡 UX-3 백그라운드 알림
→ 위젯 닫혀도 게임 클라이언트 푸시 알림 (게임 SDK가 hook 제공해야 함). 미지원 게임은 위젯 재진입 시 표시.

### 🟡 UX-4 AI 초안 보존
→ **보존** (감사·학습). 운영자 미사용 초안도 90일.

### 🟡 UX-5 변수 자동완성
→ 표준 변수 10여 개 라이브러리: `{{user.name}}`, `{{user.email}}`, `{{payment.id}}`, `{{payment.amount}}`, `{{payment.date}}`, `{{ticket.id}}`, `{{tx.hash}}`, `{{tx.status}}`, `{{character.name}}`, `{{guild.name}}`.

→ 게임 클라이언트가 prop 전달 우선. 미전달 시 시스템 `prefers-color-scheme`.

### 🟡 UX-7 CSAT 응답률 보강
→ 응답률 < 20%이면 게임 내 소량 보상 (예: 게임 화폐 10) 인센티브. 의무화 X.

### 🟡 UX-8 Discord 메시지 형식
→ 슬래시 명령 `/cs`로 진입, DM 스레드 자연어 대화. 답변은 임베드 형식 (citation footer 표시 명확).

### 🟡 UX-9 이메일 회신 포맷
- 제목 prefix: `[CS][{ticket_id}]`
- 시그니처: 표준 (게임명 + CS 채널)
- 인용 형식: `>` 들여쓰기, 마스킹 적용

### 🟡 UX-10 첨부 자동 수집 동의 UI
→ **위젯 첫 사용 1회**. 약관에 포함, 매번 X.

### 🟡 UX-11 게임 일시정지
→ **게임사 선택** (기본 일시정지 안 함). 게임 SDK가 hook 제공.

### 🟡 UX-12 익명 입력 정보
→ **닉네임 (필수) + 이메일 (선택)**. 이메일 입력 시 응답 메일 발송 가능.

### 🟡 AIP-5 "사람과 대화" 버튼 노출 룰
→ **항상 표시** (위젯 하단 고정). N턴 후 노출 정책 ❌. **단, 상담사 없으므로 문구 변경**: "사람 검토 요청" (즉시 응답 불가 안내 포함).

## 3. AI 정책

### 🟡 OQ-9 PII 마스킹 정책 (어떤 필드)
→ `PII_MASKING.md` §1 분류 그대로 채택.
- 강한 PII (전체 마스킹): 인증 비밀·지갑 시드·카드번호·주민번호·여권 등
- 중간 PII (부분 마스킹): 이메일·전화·주소·지갑 주소·tx 해시·IP
- 약한 PII (마스킹 X, 로그 별도): **캐릭터명·길드명·게임 닉네임** (AIP-2 / PII-4 동일 결정)

### 🟡 AIP-1 공감 표현 풀
→ **LLM 즉석 생성** (사전 풀 미작성). 운영 30일 후 자주 사용 표현 정리해서 풀로 전환.

### 🟡 AIP-2 / PII-4 캐릭터·길드명 PII 분류
→ **약한 PII**. 마스킹 안 함. 로그·KB·집계에는 포함 가능.

### 🟡 AIP-4 Opus 자동 트리거
→ 안전 민감 키워드 (자해·법적·위협) 자동 트리거. 어뷰징·일반 = Sonnet.

### 🟡 AIP-6 위기 핫라인
→ 우선 4개국 자체 데이터: 한국 1393 / 미국 988 / 일본 0570-064-556 / 영국 116-123. 그 외 국가 LLM이 안내 (출처 명시, 사용자 본인 검증 권장).

### 🟡 AIP-7 미지원 언어 fallback
→ **영어 응답 + 사람 큐 적재**. 사용자에게 "현재 X어 자동 응답 지원 안 함, 영어 안내 + 추후 처리" 통지.

## 4. KB / 콘텐츠 운영

### 🟡 OQ-4 / ONB-1 / KB-1 KB 도구
→ **마크다운 직접 작성 + Git 저장**. 가장 빠른 출시. 추후 Hub Builder UI 가능 시 마이그레이션.

### 🟡 OQ-10 / ONB-4 / KB-2 다국어 KB 운영
→ **CROSS 레이어 = 영어 원본 + 공식 번역** (외부 번역사 + 사람 검수).
→ **Tenant 레이어 = tenant 자율**. 영어 + 자동 번역 권장, **정책 문서만 직접 번역 필수**.

### 🟡 ONB-2 외부 import (PDF/Confluence 등)
→ **M1 미지원**. tenant가 직접 마크다운 변환.

### 🟡 ONB-3 기획서 요약
→ **tenant 운영팀**이 작성. AI 요약 도구 보조 가능 (검수 필수).

### 🟡 ONB-5 KB 작성 책임자
→ **tenant 자율**. CROSS는 강제 안 함.

### 🟡 ONB-6 시드 문항 작성 (sandbox)
→ **CROSS 운영팀** (cross_admin) 작성.

### 🟡 ONB-8 어조 프로파일 라이브러리
→ 초기 5종: **캐주얼 / 격식 / 판타지 RPG / SF / 코믹**. 추가는 tenant 요청 시.

### 🟡 ONB-9 만료 archive vs 영구 보존
→ 시즌·이벤트 종료 후 **6개월 archive** (분쟁 가능성). **게임 종료 시 7년**.

### 🟡 ONB-10 검수 미수행 패널티
→ **90일 미검수 자동 경고, 180일 priority 하향 (검색 후순위)**, 365일 비활성.

### 🟡 ONB-11 콘텐츠 라이센스 (cross-tenant 학습)
→ tenant 콘텐츠는 cross-tenant 학습 **사용 안 함**. 단 `consent.cs_ai_training` 별도 동의 시 가능.

### 🟡 ONB-12 시즌 종료 트리거
→ **운영자 수동 + tenant 캘린더 입력 양쪽 지원**.

### 🟡 ONB-13 Sandbox 비용 부담
→ **CROSS 부담** (초기 출시 인센티브).

### 🟡 ONB-14 hyper-care 운영
→ 출시 후 **첫 2주 cross_admin이 24시간 모니터링 알람만 수신**.

### 🟡 ONB-15 외부 인용 (게임 위키 등)
→ **M1 금지**. 자체 작성 콘텐츠만.

### 🟡 KB-3 정책 법무 검토 워크플로
→ **외부 (이메일/Slack 워크플로)**. KB 도구 통합은 출시 후 검토.

### 🟡 KB-5 disambiguation
→ `disambig` 메타 필드. 예: "보린 (캐릭터)" vs "보린 (길드)" / "아리아 (캐릭터)" vs "아리아 (NPC)".

### 🟡 KB-6 패치노트 자동 import
→ **M1 미지원**.

### 🟡 KB-7 KB 길이 한도
→ **한 문서 5,000자 권장 상한**. 초과 시 분해 권장 (강제 X).

### 🟡 KB-8 시즌 자동 비활성 트리거
→ `valid_until` 메타 + tenant 캘린더 둘 다.

### 🟡 KB-9 분쟁 진행 중 정책 변경
→ **분쟁 발생 시점 정책 유지**. 변경 후 발생 케이스만 새 정책 적용.

### 🟡 CKB-1 초기 KB 최소 임계
→ **카테고리당 5개 이상** (총 25개 이상).

### 🟡 CKB-2 시드 노출 자동 동결
→ **자동 동결 X** (안내·교육만). 자산 이체는 유저가 직접 수행 (지갑 비수탁성).

### 🟡 CKB-3 위기 핫라인 데이터 출처
→ AIP-6과 동일. 4개국 자체 + 그 외 LLM 안내.

### 🟡 CKB-4 정책 vs FAQ 톤
→ **정책 = 법무 검토 문구 그대로**, **FAQ = 운영팀이 풀어 씀**.

### 🟡 CKB-5 신규 체인 KB 작성자
→ **cross_infra 운영팀**.

### 🟡 CKB-6 번역 운영
→ **외주 번역사 + 사람 검수** (자동 번역 단독 X).

### 🟡 CKB-7 모순 우선순위 (tenant vs CROSS)
→ **tenant 레이어 우선** (자기 게임 정책 우선). 단 **CROSS 보안·법무·온체인 기술 정책은 강제 (override 불가)**. Q-011-B 확정.

### 🟡 CKB-9 결제 시스템 차이
→ tenant 레이어로 별도 작성. CROSS 레이어는 일반 안내만.

### 🟡 CKB-10 Safety 카테고리 보존
→ **7년** (분쟁 시 동일).

## 5. 신원·권한 (RBAC)

### 🟡 OQ-8 Discord 봇 권한·매핑
- **봇 권한**: `applications.commands` + `bot` scope, `Send Messages` + `Embed Links` + `Read Message History` + `Attach Files` 권한
- **매핑**: Discord OAuth 1회 링크 → `discord_user_id` ↔ Profile (Profile Store 내 별도 테이블)

### 🟡 OQ-16 anonymous 변경 액션
→ **절대 금지**. 본인 인증 우선 안내.

### 🟡 RBAC-2 tenant_admin KB의 cross_admin 검토
→ **필수 X** (tenant 자율). 단 CROSS 정책과 모순 시 자동 경고.

### 🟡 RBAC-3 JIT 권한 (사람 vs 자동)
→ **자동 룰** (사유 입력 → 자동 발급, 사후 감사). cross_admin 권한 발급만 사람 승인.

### 🟡 RBAC-4 MFA 강제 범위
→ **모든 CROSS 역할 + tenant_admin 필수**. tenant_operator 권장 (변경 액션 권한 보유 시 필수). 방식은 DEV/INFRA.

### 🟡 RBAC-5 외주·임시 계정 유효기간
→ **최대 90일**, 갱신 가능 (사유 + 승인).

### 🟡 RBAC-6 raw view 임계 N
→ **일 50건**. 초과 시 보안팀 알람.

### 🟡 RBAC-7 developer JIT 자동 발급 사유
→ 3개 사유로 한정: **디버깅 / 인시던트 대응 / 고객 지원**.

### 🟡 RBAC-8 피신고자 정보 승인 SLA
→ 운영시간 **4시간**, 야간 8시간. 상담사 없음으로 cross_admin 직접 처리.

### 🟡 RBAC-9 다국가 법규 우선순위
→ **거주 국가 우선, 충돌 시 GDPR 기준** (가장 엄격).

### 🟡 RBAC-10 역할 세분
→ **M1은 현재 11개로 출시**. tenant_billing/tenant_dev 등 세분은 운영 데이터 보고 결정.

## 6. PII / 컴플라이언스

### 🟡 PII-5 OCR 후 원본 보존
→ **첨부 이미지 원본 18개월** (분쟁 7년). OCR 텍스트는 마스킹 후 로그.

### 🟡 PII-7 외부 감사 주기·인증
→ 출시 후 **6개월 내 SOC 2 Type II 권장 목표**. 한국 사업 본격화 시 ISMS-P.

### 🟡 PII-8 국가별 보존 차이
→ **GDPR 기준 우선** (가장 엄격). 한국 PIPA 의무 케이스만 별도. 자세한 매트릭스는 법무 검토.

### 🟡 PII-9 사고 대응 SLA
→ GDPR **72시간** (감독기관) / 한국 PIPA **24시간** (당사자 통보) / **72시간** (감독기관).

### 🟡 CMP-2 Discord 동의 흐름
→ 봇 첫 인입 시 **봇 메시지 + 외부 동의 페이지 링크**.

### 🟡 CMP-3 이메일 동의 미완료 처리
→ **자동 회신 + 동의 링크 발송**. 동의 전엔 일반 안내만 가능 (변경 액션·PII 처리 X).

### 🟡 CMP-4 부분 삭제 정합성
→ **분쟁 진행 중 데이터는 부분 삭제 거부**, 사유 회신 + 분쟁 종결 후 재요청 안내.

### 🟡 CMP-5 익명화 기준
→ **k-anonymity k=5** (PII 제거 + 식별 가능 조합 5인 이상 보장).

### 🟡 CMP-6 거주 국가 판정
→ **가입 국가 + IP**. 충돌 시 가장 엄격한 규제 적용.

### 🟡 CMP-7 사고 분류 임계
- **Critical**: 100건 이상 PII 영향 OR PII 1건이라도 외부 노출 OR 시드·카드번호 1건 노출
- **High**: 10~99건 PII 영향
- **Medium**: 1~9건 PII 영향, 외부 노출 없음
- **Low**: 마스킹 누락 발견 (실제 노출 없음)

### 🟡 CMP-8 Subprocessor 변경 공지
→ **개인정보 처리방침 페이지 업데이트 + 옵트인 사용자 이메일 알림**. 30일 사전.

### 🟡 CMP-9 미성년 트리거
→ **가입 정보 + 결제 한도 + 대화 정황** (자해·미성년 정황 키워드 감지).

### 🟡 CMP-10 PCI DSS
→ **결제 위임 (Stripe 등)** → CROSS는 적용 X.

### 🟡 CMP-11 CCPA "판매 거부"
→ **광고 분석에 사용 안 함** → 적용 X. 단 개인정보 처리방침에 "데이터 판매 안 함" 명시.

### 🔴 CMP-1 동의 항목 약관 작성 — **법무 작성 필요**
### 🔴 CMP-12 약관 다국어 번역 — **법무 + 외주 번역사**
### 🔴 EP-6 다국가 법무 통보 매핑 — **법무 작성 필요**
