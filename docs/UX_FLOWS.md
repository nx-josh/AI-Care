# 화면·UX 플로우 (Product-level)

**문서 상태**: v0.1 초안
**최종 수정일**: 2026-05-19
**관계 문서**: `PRD.md` §5 (채널), §6 (핵심 플로우), `AI_POLICY.md` §2 (응답 구조), `TENANT_ONBOARDING.md`
**대상 검토자**: 디자이너 / PM / CS 운영팀 / 게임사 통합 담당자

본 문서는 유저·운영자·tenant_admin 각 표면의 **product-level UX 플로우**. 와이어프레임 수준 묘사로, 구현 디테일·디자인 시스템 토큰은 별도 트랙.

---

## 0. 핵심 원칙

1. **위젯은 게임 일부처럼** — 별창·새 탭 ❌. 인게임 위 오버레이.
2. **3초 안에 도달 가능** — 위젯 호출 → 첫 입력까지 3초.
3. **"사람과 대화"는 항상 보임** — N턴 후 노출 같은 페널티 없음.
4. **출처는 답변에 따라간다** — citation footer 항상 표시, 클릭해서 원문 확인 가능.
5. **상태 명확** — AI 응답 중 / 사람 대기 중 / 종결 — 사용자가 항상 인지.
6. **다국어 UI** — 화면 텍스트도 입력 언어 따라감.
7. **접근성** — 키보드 / 스크린 리더 / 색맹 / 큰 글씨.

## 2. 인게임 위젯 (P0 surface)

### 2.1 진입 트리거
게임 클라이언트가 위젯을 띄우는 시점:
- 메인 메뉴 "고객센터" 버튼
- 결제·민팅 실패 직후 자동 호출 (선택 옵션)
- 게임 내 알림(이벤트 종료 등)에서 "문의" 링크
- 인게임 핫키 (선택)

### 2.2 첫 화면 — Welcome
```
┌─────────────────────────────────┐
│  ← 닫기                          │
│                                 │
│  안녕하세요! 어떤 도움이 필요하세요? │
│                                 │
│  자주 묻는 질문:                  │
│  [ 결제 / 환불 ]                 │
│  [ 게임 진행 ]                   │
│  [ 버그 신고 ]                   │
│  [ 다른 유저 신고 ]              │
│                                 │
│  [ 입력하기 ↓                  ] │
│                                 │
│  ─────────────────────────      │
│  [💬 바로 사람과 대화]            │
└─────────────────────────────────┘
```

- **빠른 옵션 칩** — 카테고리별 진입 (분류기 부담 ↓).
- **자유 입력** — 칩 무시하고 그냥 쓰기.
- **"사람과 대화"** — 항상 하단 고정.

### 2.3 대화 진행
```
┌─────────────────────────────────┐
│  ← 결제 / 환불                   │
├─────────────────────────────────┤
│                                 │
│  5월 10일 49.99달러 결제         │
│  환불해주세요         [09:31, 나] │
│                                 │
│  ──                             │
│  [🤖 처리 중...]                 │
│  ──                             │
│  확인했습니다. 5월 10일 49.99 USD │
│  결제 내역 조회됐어요. 환불은     │
│  결제팀이 직접 처리합니다.        │
│  24시간 내 회신 드릴게요.        │
│                                 │
│  📎 [출처: CROSS Refund Policy   │
│       v1.2 §3.1]                │
│                  [09:31, AI]    │
│                                 │
│  💬 결제팀 대기 중 (24시간 이내)   │
├─────────────────────────────────┤
│  [ 추가 메시지 입력         ↑ ] │
│  [📎 첨부] [💬 사람과 대화]       │
└─────────────────────────────────┘
```

- AI 응답 footer에 **출처 인용** 클릭 가능.
- 에스컬레이션 발생 시 **상태 배너** ("결제팀 대기 중") 표시.
- 추가 메시지 입력 가능.

### 2.4 첨부 흐름
```
[ 📎 첨부 클릭 ]
   │
   ▼
┌────────────────────┐
│  📷 사진 촬영       │
│  🖼️ 라이브러리       │
│  📄 로그 첨부       │ ← 게임 클라이언트가 자동 수집
│  🎥 동영상          │
└────────────────────┘
```

- 게임 클라이언트가 디바이스·세션 컨텍스트(빌드·OS·시각) **자동 첨부** (TENANT_ONBOARDING T1, 유저가 인지하도록 표시).
- 첨부 후 미리보기 + 삭제 가능.
- 파일 크기 한도 도달 시 안내.

### 2.5 "사람과 대화" 전환
유저가 누른 즉시:
- AI는 답변 멈춤.
- "잠시만요. 사람 상담사 연결 중입니다. (예상 응답: 24시간 이내)" 안내.
- 위젯은 그대로 유지 (유저가 닫으면 백그라운드에서 처리).

### 2.6 응답 도착 알림
사람 응답이 오면:
- 게임 내 푸시 알림 (선택 옵션)
- 위젯 재진입 시 새 응답 highlighted
- 이메일·Discord 채널 사용자는 해당 채널로 자동 회신

### 2.7 종결 / 만족도
```
┌─────────────────────────────────┐
│  ← 결제 / 환불                   │
├─────────────────────────────────┤
│  ...                            │
│                                 │
│  환불 처리 완료됐어요. 영업일     │
│  기준 3~5일 내 카드사 반영       │
│  예정입니다.    [11:45, 결제팀]  │
│                                 │
│  이번 응대에 얼마나 만족하셨나요? │
│  ⭐ ⭐ ⭐ ⭐ ⭐                  │
│  [코멘트 (선택)               ] │
│                                 │
│  [ 평가 제출 ]                  │
│  [ 다시 도움 필요해요 ]          │
└─────────────────────────────────┘
```

- CSAT 1-5점 + 코멘트 (KPI §7.1).
- "다시 도움 필요" 누르면 재오픈.
- 종결 30일 이내 무응답이면 자동 closed.

### 2.8 에러·오프라인 상태
- 네트워크 끊김 → "연결 끊김. 자동 재시도 중..."
- AI 일시 장애 → "잠시 후 사람 상담사가 도와드릴게요" + 자동 사람 라우팅
- 로그인 안 됨 → 익명 안내 + "본인 인증 후 상세 처리 가능"

## 3. Hub Console (운영자 표면)

### 3.1 로그인·큐 선택
- SSO + MFA (RBAC §7.4)
- 역할에 따라 보이는 큐 다름:
  - `tenant_operator` → 자기 tenant 큐만
  - `cross_payment` → 결제 큐
  - 등등

### 3.2 티켓 리스트
```
┌────────────────────────────────────────────────┐
│  Hub Console — CROSS 결제 큐                   │
│  [필터: priority▾] [카테고리▾] [언어▾] [검색]  │
├────────────────────────────────────────────────┤
│  ⚠ high  ⏰ 2h ago    Pixel Heroes / 결제 누락 │
│  • normal  ⏰ 5h ago   Empire Wars / 환불 요청 │
│  • normal  ⏰ 1d ago   Pixel Heroes / 영수증 분쟁│
│  ✓ resolved  ⏰ 3d ago Magic Cards / 환불 완료 │
│  ...                                           │
└────────────────────────────────────────────────┘
```

- 우선순위 색상 (high=빨강, normal=중립, low=회색)
- 시간 표시 (몇 시간 전 / 며칠 전)
- tenant 이름 표시 (cross_* 역할에 한정)
- 한 행 클릭 → 상세

### 3.3 티켓 상세 (3단 레이아웃)
```
┌──────────────┬───────────────────┬──────────────┐
│ [좌] 대화 패널│ [중] 작성 패널     │ [우] 컨텍스트 │
│              │                   │              │
│ 유저:        │ AI 초안 (수정 가능)│ 프로파일      │
│ "환불해..."  │ ┌──────────────┐  │ - 이름        │
│              │ │ 확인했습...   │  │ - 가입일       │
│ AI:          │ │              │  │ - 밴 여부      │
│ "확인했..."  │ └──────────────┘  │              │
│              │ [수정] [그대로 발송]│ 최근 결제     │
│ 유저:        │                   │ - 5/10 $49.99 │
│ "빨리..."    │ [✏️ 직접 작성]    │   refunded    │
│              │                   │              │
│              │ [📎 첨부] [🔀 큐]  │ 툴 호출 로그  │
│              │ [✅ resolve]      │ - payment_... │
│              │                   │ - escalate_...│
│              │                   │              │
└──────────────┴───────────────────┴──────────────┘
```

- **좌**: 대화 전체 시계열
- **중**: AI 초안 + 운영자 작성 도구
- **우**: 프로파일·결제·툴 로그·유사 과거 티켓

### 3.4 AI 초안 검토 패널
- AI 초안 그대로 보낼 거면 "그대로 발송" 한 번 클릭.
- 수정 필요 시 인라인 편집 → "수정 발송".
- AI 무시하고 새로 쓰려면 "직접 작성" 토글.

### 3.5 응답 작성·발송
- 마크다운 지원 (간단한 굵게·기울임·링크).
- 변수 삽입 (예: `{{user.name}}`, `{{payment.id}}`) — 프로파일에서 자동 인용.
- 송신 전 PII 검사 한 번 더 (PII_MASKING §3.3 출력 검증).

### 3.6 에스컬레이션·큐 이동
- 큐 변경 버튼 → 큐 선택 + 사유 입력 → 핸드오프.
- 새 큐 담당자에게 알림 자동 발송.

### 3.7 검색·필터
- 유저 ID / 키워드 / 카테고리 / 기간 / 상태 / 채널.
- tenant 측 역할은 자기 tenant만, CROSS 역할은 자기 큐 범위만.

### 3.8 메트릭 대시보드
- 자기 큐 / tenant 메트릭 (PRD §7).
- 자동 해결률·응답 시간·에스컬레이션 비율 시계열.
- 운영자 본인 처리 통계 (선택 — 개인정보 우려로 옵트인).

## 4. KB 작성 도구

### 4.1 메타데이터 입력
```
┌─────────────────────────────────┐
│  새 KB 문서 작성                 │
├─────────────────────────────────┤
│  제목         [_____________]   │
│  슬러그       [auto: from-title]│
│  카테고리     [▾ FAQ          ] │
│  언어         [▾ 한국어        ] │
│  버전         [v1.0]            │
│  유효 시작    [2026-05-19]      │
│  유효 종료    [선택]            │
│  태그         [+추가]           │
│  출처         [기획서 §3.2]     │
└─────────────────────────────────┘
```

### 4.2 본문 작성
- 마크다운 에디터 (실시간 미리보기 분할 화면).
- "한 문서 = 한 답변 단위" 가이드라인 inline 표시.
- 길이 카운터.

### 4.3 검수 결과 표시
```
┌─────────────────────────────────┐
│  자동 검수 결과                  │
├─────────────────────────────────┤
│  ✅ PII 스캐너         통과       │
│  ✅ Citation 메타데이터  완전     │
│  ✅ 언어 일치           통과     │
│  ⚠️ 모순 감지         CROSS 환불 │
│      정책과 부분 충돌 — 확인 필요 │
│  ✅ 중복 검사           통과     │
│  ✅ 길이               1,200자  │
│                                 │
│  [수정] [그대로 제출]            │
└─────────────────────────────────┘
```

- 통과 항목은 초록, 경고는 노랑, 실패는 빨강.
- 클릭하면 상세 사유 표시.
- 실패 항목 있으면 "제출" 비활성화.

### 4.4 승인·발행 흐름
```
[작성] → [자동 검수] → [승인 요청] → [tenant_admin 검토]
                                       │
                                       ▼
                              [승인] → [인덱싱] → [활성]
                              [반려] → [작성자에게 회신]
```

- 작성자 ≠ 승인자 (SoD).
- 승인 알림은 사내 채널 (Slack 등).
- 인덱싱 lag < 1시간 (PRD §7.2).

## 5. 임베드 가이드 (게임 클라이언트 통합)

### 5.1 통합 흐름 (게임 개발자 관점)
1. CROSS 계정 생성 → tenant_id 발급
2. SDK 다운로드 (게임 엔진별: Unity / Unreal / Web)
3. 인증 토큰 설정 (Firebase 또는 Game UUID + HMAC 키)
4. 위젯 호출 API 1줄 — `cross.cs.open()`
5. (선택) 컨텍스트 자동 첨부 설정 (디바이스·세션·빌드·로그 hook)
6. (선택) 트리거 위치 디자인 — 메인 메뉴 / 결제 실패 / 등등

### 5.2 호출 시퀀스 (개념)
```
[게임 클라이언트]                [CROSS Backend]
       │                              │
       │  open() 호출                 │
       │ ─────────────────────────▶  │
       │                              │ tenant_id, profile 검증
       │  위젯 URL + 토큰              │
       │ ◀───────────────────────── │
       │                              │
       │  WebView 또는 네이티브 UI 띄움 │
       │                              │
       │  유저 대화 (POST /agent/chat) │
       │ ◀────────────────────────▶ │
```

### 5.3 표시 트리거
- 직접 호출 (`open()`)
- 이벤트 후크 (결제 실패·민팅 실패 후 자동 prompt)
- Deep link (이메일·푸시에서 위젯 직접 진입)

## 6. 다국어 UI

- 위젯·Hub Console·KB 도구의 화면 텍스트 모두 다국어 지원.
- 위젯: 입력 언어 자동 감지에 따라 UI 텍스트도 매칭.
- Hub Console·KB 도구: 운영자 설정 언어 (개인 프로필).
- 우측 정렬 언어(아랍어 등): RTL 레이아웃 자동 전환.

## 7. 접근성 (a11y)

| 항목 | 요구 |
|---|---|
| 키보드 네비게이션 | 모든 액션 키보드로 가능 |
| 스크린 리더 | 채팅 메시지·상태 변화 announce |
| 색맹 | 우선순위·상태를 색만으로 구분하지 않음 (아이콘·텍스트 병행) |
| 글씨 크기 | 시스템 설정 따름, 200%까지 확대 시 깨지지 않음 |
| 대비 | WCAG AA 이상 |
| 자막·대체 텍스트 | 첨부 이미지 alt, 동영상 자막 권장 |
