# 컴플라이언스 (Consent · 데이터 삭제 · 규제)

**문서 상태**: v0.1 초안
**최종 수정일**: 2026-05-19
**관계 문서**: `PII_MASKING.md`, `RBAC_MATRIX.md`, `AI_POLICY.md`, `TENANT_ONBOARDING.md`
**대상 검토자**: DPO / 법무 / 보안팀 / cross_admin

본 문서는 CROSS AI Care 시스템의 **법적·규제 요구사항**과 운영 절차를 정리. GDPR(EU), CCPA(미국 캘리포니아), PIPA(한국), APPI(일본), LGPD(브라질) 등 주요 규제 매트릭스 + 동의 흐름 + 데이터 삭제 절차.

---

## 0. 핵심 원칙

1. **거주 국가 규제 우선** — 유저 거주 국가의 가장 엄격한 기준 적용.
2. **명시적 동의** — 묵시적·디폴트 동의 금지.
3. **동의 분리** — 필수·선택 동의를 별 축으로 관리 (`consent.cs_basic` ≠ `consent.cs_ai_training`).
4. **삭제 권리 존중** — 법적 보존 의무가 없는 한 30일 내 처리.
5. **사고 통보 의무** — 법정 기간 내 감독기관·당사자 통보.
6. **감사 가능** — 모든 동의·삭제·접근 이력 보존.

## 1. 동의(Consent) 흐름

### 1.1 동의 항목

| ID | 설명 | 필수/선택 | 동의 시점 |
|---|---|---|---|
| `consent.cs_basic` | CS 문의 처리 기본 데이터 수집·이용 | 필수 | 위젯 첫 사용 / 채널 첫 인입 |
| `consent.cs_ai_training` | 본인 데이터 AI 학습용 활용 | 선택 | 동상 (별도 옵트인) |
| `consent.cs_marketing` | CS 채널 통한 마케팅 안내 | 선택 | 동상 |
| `consent.cs_cross_tenant_analytics` | 익명 집계 통계에 본인 데이터 포함 | 선택 | 동상 |

### 1.2 채널별 동의 흐름

#### SDK 위젯
- 위젯 첫 사용 시 모달 표시: 필수 동의 + 선택 동의 체크박스.
- 필수 미동의 시 위젯 사용 불가, "동의 후 사용해주세요" 안내.
- 선택 동의는 언제든지 설정에서 변경 가능.

#### Discord
- Discord 봇 첫 DM 인입 시 자동 안내 메시지 + 동의 링크.
- 동의 페이지에서 처리 후 봇 사용 활성화.

#### 이메일
- 첫 인입 메일에 대한 자동 회신에 동의 안내·링크.
- 동의 미완료 시 일반 안내만 가능, 변경 액션·PII 처리 불가.

### 1.3 동의 철회
- 사용자 설정에서 언제든지 철회 가능.
- 철회 시점부터 **미래 처리에만 적용**. 과거 데이터는 보존 의무에 따름.
- 철회 후 30일간 미사용 → 안내 메시지 ("CS 사용 시 재동의 필요").

### 1.4 미성년자 동의
| 연령 | 처리 |
|---|---|
| **만 14세 미만 (한국)** | 보호자 동의 필수 |
| **만 13세 미만 (미국 COPPA)** | 동상 |
| **만 16세 미만 (EU GDPR 옵션)** | 국가별 차이, 가장 엄격 적용 |

- 게임사가 연령 정보 보유 시 자동 분기.
- 미보유 시 가입·첫 사용 시 연령 확인 단계.
- 미성년자는 자동으로 `consent.cs_ai_training` 비활성 (옵트아웃 디폴트).

### 1.5 동의 기록
- 모든 동의·철회 이력 영구 보존 (감사 목적).
- 항목별·시각·IP·디바이스 기록.
- 동의서 버전(`v1.2` 등) 기록 → 약관 변경 시 재동의.

§PII_MASKING.md §6과 동일. 요약:

|---|---|

### 2.1 국가별 보존 정책 충돌

- 한국(PIPA): 5년 보존 의무 케이스 있음.
- 미국(CCPA): 명시 보존 기간 없음.

**적용 룰**: 거주 국가의 가장 엄격한 의무 우선. 모순 시 법무 검토.

## 3. 데이터 삭제 요청 (Right to Erasure)

### 3.1 요청 채널
- CS 위젯 → "데이터 삭제 요청" 메뉴 → DPO 큐로 자동.
- 이메일 (dpo@... 또는 게임사 dpo 주소) → 동일 큐.
- 외부 양식 (개인정보 처리방침에 명시).

### 3.2 처리 절차
1. **수신 확인** — 24시간 내 자동 회신 (접수 안내).
2. **본인 확인** — 신분증·계정 인증 (이메일 토큰·OTP).
3. **법적 보존 의무 확인** — 분쟁 진행·법정 의무 여부.
4. **삭제 실행**:
   - 보존 의무 없음 → 30일 이내 삭제 (격리 저장소 + 백업).
   - 보존 의무 있음 → 거절 사유 회신 (사유 명시).
5. **완료 통지** — 처리 완료 회신.

### 3.3 부분 삭제
- 일부 데이터만 삭제 요청 가능 (예: 결제 이력만 보존, 대화는 삭제).
- 단, 정합성 깨질 우려 시 안내 (예: 결제 분쟁 진행 중인데 대화 삭제 요청 → 보류 안내).

### 3.4 익명화 (대안)
- 완전 삭제 대신 익명화 가능 (분석·통계 활용).
- 익명화 기준: PII 제거 + 식별 가능 조합 제거 (k-anonymity 또는 동급).

## 4. 다국가 규제 매트릭스

### 4.1 주요 규제

| 규제 | 적용 국가 | 핵심 요구 |
|---|---|---|
| **GDPR** | EU + EEA | 명시적 동의·삭제 권리·이동 권리·DPO 의무 |
| **CCPA / CPRA** | 미국 캘리포니아 | 판매 거부·접근·삭제·차별 금지 |
| **PIPA** | 한국 | 개인정보 처리 동의·고유식별정보 별도 동의·국외 이전 동의 |
| **APPI** | 일본 | 처리 목적·제3자 제공 동의·국외 이전 |
| **LGPD** | 브라질 | DPO 의무·동의·삭제·이동 |
| **PIPL** | 중국 | 명시 동의·국외 이전 별도 평가 |
| **COPPA** | 미국 (13세 미만) | 보호자 동의·수집 최소화 |

### 4.2 핵심 차이 항목

| 항목 | GDPR | CCPA | PIPA | APPI |
|---|---|---|---|---|
| 동의 방식 | 명시·자유로움 | 옵트아웃 가능 | 명시 | 명시 |
| 삭제 권리 | 있음 | 있음 | 있음 | 있음 (조건부) |
| 이동 권리 | 있음 | 있음 | 부분적 | 부분적 |
| 사고 통보 기한 | 72시간 | 즉시 (불합리한 지연 X) | 24시간 (당사자) / 72시간 (감독기관) | 즉시 |
| DPO 의무 | 조건부 | X | 조건부 | X |
| 미성년자 | 16세 (국가별) | 13세 | 14세 | 18세 (보호자 동의) |
| 국외 이전 | 적정성 결정·SCC | — | 동의 | 동의·기준 적합 |

### 4.3 운영 룰
- **거주 국가 기준** — 유저 거주 국가 (가입 정보 또는 IP 기반).
- **충돌 시 가장 엄격** — 보존·동의·삭제 등에서 더 엄격한 기준.
- **새 국가 추가 시** — 법무 검토 필수, 본 매트릭스 갱신.

## 5. 미성년자 처리 (확장)

### 5.1 연령 확인
- 게임사가 가입 시 연령 확인 책임 1차.
- CROSS AI Care 측은 게임사 제공 정보 신뢰 (별도 확인 안 함).
- 단, 대화 중 미성년 정황 발견 → 안전 모드 전환 + 보호자 안내.

### 5.2 미성년자 안전 모드
| 항목 | 처리 |
|---|---|
| `consent.cs_ai_training` | 옵트아웃 디폴트 |
| 결제 변경 액션 | 보호자 동의 추가 필요 |
| 모더 신고 | 일반 + 미성년 안전팀 동시 라우팅 |
| 안전 민감 (자해 등) | 미성년 위기 핫라인 별도 |
| 광고·마케팅 동의 | 비활성 |

## 6. Subprocessor (외주·해외 처리자)

### 6.1 정의
- CROSS AI Care 서비스 제공에 데이터 처리에 관여하는 제3자.
- 예: LLM 제공자 (Anthropic), 클라우드 (AWS/GCP), 이메일 (Resend/Postmark), 번역 외주.

### 6.2 공시
- 개인정보 처리방침에 subprocessor 목록·국가·처리 목적 공시.
- 변경 시 30일 사전 공지 (가능한 경우).

### 6.3 DPA (Data Processing Agreement)
- 모든 subprocessor와 DPA 체결 필수.
- 표준 계약 조항(SCC) 또는 적정성 결정 활용 (국외 이전).
- 정기 검토 (연 1회).

## 7. 사고 통보 (Data Breach Notification)

### 7.1 사고 분류
| 분류 | 예 |
|---|---|
| **Critical** | PII 대량 유출, 외부 침해 |
| **High** | 단일 PII 누설, 권한 위반 |
| **Medium** | 마스킹 누락, RAG 인덱스 PII |
| **Low** | 내부 접근 위반 (외부 노출 없음) |

### 7.2 통보 의무 (요약, 정확한 기한은 법무 확인 필수)

| 규제 | 감독기관 통보 | 당사자 통보 |
|---|---|---|
| GDPR | 72시간 | 위험 고지 후 지체없이 |
| PIPA | 72시간 | 24시간 (위험 시) |
| CCPA | 즉시 | 즉시 |
| APPI | 즉시 | 즉시 |

### 7.3 사고 대응 워크플로
1. **탐지** → 보안팀 즉시 인지.
2. **초기 봉쇄** → 영향 범위 격리, 추가 노출 차단.
3. **영향 평가** → 영향 받은 유저 수·데이터 유형·위험도.
4. **법적 통보** → 법무·DPO 협의, 기한 내 감독기관·당사자 통보.
5. **재발 방지** → 원인 분석, 정책·시스템 보강.
6. **사후 보고** → 7년 보존 (인시던트 보고서).

## 8. 감사·인증

### 8.1 내부 감사
- 분기 1회 보안팀 + DPO 합동 감사.
- 항목: 동의 기록·접근 로그·삭제 처리·마스킹·인시던트.

### 8.2 외부 감사 (선택)
| 인증 | 적용 가능성 | 비고 |
|---|---|---|
| **SOC 2 Type II** | 권장 | B2B 신뢰성 |
| **ISO 27001** | 권장 | 정보 보안 관리 |
| **ISMS-P** | 한국 사업 시 | 한국 인증 |
| **PCI DSS** | 결제 처리 직접 시 | CROSS는 통상 결제 사업자 거치므로 적용 X 가능 |

### 8.3 인증 우선순위
- M1 출시 이후 ~ 6개월 내 SOC 2 Type II 권장.
- 한국 사업 본격화 시 ISMS-P.
