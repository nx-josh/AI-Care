# KB 작성 가이드

**문서 상태**: v0.1 초안
**최종 수정일**: 2026-05-19
**관계 문서**: `TENANT_ONBOARDING.md` (전체 플로우), `PII_MASKING.md` (PII 차단), `AI_POLICY.md` §4 (Citation 포맷)
**대상**: `tenant_operator` (KB 작성자) / `tenant_admin` (승인자)

본 문서는 tenant 측 운영자가 **자기 게임의 KB를 어떻게 작성해야 AI가 잘 인용·답변할 수 있는지**의 실전 가이드. 검수 통과 + 인용 가능 + PII 안전 — 세 가지 원칙으로 정리.

---

## 0. 핵심 원칙 5가지

1. **한 문서 = 한 답변 단위** — 인용 가능한 크기로 분해.
2. **출처가 명확해야 인용된다** — 제목·버전·섹션·태그 빠뜨리지 말 것.
3. **PII는 절대 본문에 포함 X** — 작성 시점에 시스템이 막아도, 작성자가 1차 방어.
4. **확정 표현보다 조건 명시** — "환불됩니다" ❌, "다음 조건일 때 환불 가능합니다" ✅.
5. **유저가 쓰는 말로 써라** — 내부 코드명·약어 ❌, 게임 내 표기 그대로 ✅.

## 1. 좋은 KB의 5가지 특징

| 특징 | 설명 | 예 |
|---|---|---|
| **명확한 제목** | 유저가 검색할 만한 표현 | "5월 결제 환불 정책" (✅) / "RP-2026-05" (❌) |
| **싱글 토픽** | 한 문서가 한 주제만 | "환불 가능 조건" 따로, "환불 절차" 따로 |
| **답 위주 구조** | 결론 먼저, 부연 나중 | 첫 줄에 핵심 답, 다음 단락에 조건·예외 |
| **메타데이터 완비** | 제목·버전·섹션·태그·언어 모두 채움 | (다음 섹션) |
| **버전 추적** | 정책 변경 시 새 버전, 옛 버전 archive | "환불 정책 v1.2 → v1.3" |

## 2. FAQ 작성 패턴

### 2.1 표준 템플릿

```markdown
---
title: 길드를 탈퇴하려면 어떻게 하나요?
slug: how-to-leave-guild
category: faq
language: ko
version: v1.0
valid_from: 2026-05-19
tags: [guild, member, leave]
source: 기획서 §5.3 길드 시스템
---

길드 화면 → 본인 프로필 → [길드 탈퇴] 버튼으로 가능합니다.

탈퇴 시 다음에 유의해주세요:
- 길드 마스터는 탈퇴 전 마스터 권한을 다른 멤버에게 이양해야 합니다.
- 길드 기여 포인트는 초기화됩니다.
- 24시간 내 같은 길드 재가입은 불가능합니다.
```

### 2.2 좋은 FAQ 예시 vs 나쁜 FAQ 예시

#### ✅ 좋은 예
- **제목**: "용병 5명 모았는데 다음 퀘스트가 안 열려요"
- **본문**: 핵심 답 1문장 + 조건 2-3개 + 관련 가이드 링크

#### ❌ 나쁜 예
- **제목**: "Q12-2" (검색 안 됨)
- **본문**: 기획서 통째 복사 (인용 위치 모호)
- **본문**: "관련 가이드를 참조하세요" (답이 없음)
- **본문**: "관리자에게 문의하세요" (자동 응답 의미 무용)

### 2.3 분해 가이드
- 한 답변이 6문장 넘으면 분해 검토.
- "그리고 이건 또..." 추가 정보는 별도 FAQ로.
- 단, 너무 잘게 쪼개면 검색 매칭 안 좋아짐 — "한 의문"이 단위.

## 3. 정책 문서 작성 패턴

정책(환불·계정·ToS)은 FAQ와 달리 **법적 효력**을 가질 수 있어 더 엄격.

### 3.1 정책 표준 구조
```markdown
---
title: 결제 환불 정책
slug: refund-policy
category: account
language: ko
version: v1.2
valid_from: 2026-05-01
tags: [refund, payment, policy]
source: 운영팀 정책 문서 2026-04-30
---

# 환불 가능 조건

다음 조건을 모두 만족할 때 환불 가능합니다:
1. 결제 후 7일 이내
2. 해당 상품을 사용하지 않은 상태 (소비형 아이템 제외)
3. 결제 수단에 이상이 없는 경우

# 환불 불가 케이스
- 소비형 아이템 (이미 사용)
- 시즌 패스 구매 후 시즌 진행 중
- 자체 약관에 명시된 예외 케이스

# 환불 절차
1. 위젯 또는 이메일로 환불 요청
2. 결제팀이 24시간 내 검토
3. 승인 시 영업일 3~5일 내 환급
```

### 3.2 정책 작성 시 주의
- **확정·약속 표현 피하기** — "반드시", "100%", "즉시" 같은 단언 ❌.
- **예외 조항 명시** — "단, ..." 형식으로 예외 분리.
- **버전 변경 시 옛 버전 보존** — 분쟁 시 그 시점 정책 인용 가능해야.
- **법무 검토 거친 문구만** — 운영자가 임의로 정책 수정 ❌, 정책팀 결재 거침.

## 4. 패치노트 → KB 변환

게임 패치노트를 그대로 KB에 올리면 노이즈가 많음. **유저가 물어볼 만한 단위**로 변환.

### 4.1 변환 대상
- 신규 콘텐츠 (캐릭터·아이템·시스템) → 각각 별 KB
- 정책 변경 → 정책 문서 새 버전
- 임시 알려진 버그 → 임시 KB (해결 시 archive)
- UI 변경 → 변경 전후 비교 (선택)

### 4.2 변환 가이드
- 패치노트의 "보상" 변경 → "이번 시즌 보상은 무엇인가요?" FAQ.
- "버그 수정" 목록 → 각 버그가 외부 영향 있었으면 별도 FAQ.
- "밸런스 조정" → 영향 받는 시스템 가이드 업데이트.

### 4.3 패치노트 자체 KB
- 패치노트 한 버전은 1개 KB로 등록 (요약).
- 본문: "이번 패치(v2.5)의 주요 변경:" + 항목 5-10개.
- 상세는 각 카테고리 KB로 링크.

## 5. 캐릭터·아이템 사전

### 5.1 형식
```markdown
---
title: 아리아 (캐릭터)
slug: character-aria
category: faq
language: ko
version: v1.0
tags: [character, mage, premium]
source: 기획서 §2.1 캐릭터 도감
---

**아리아**는 마법사 클래스의 프리미엄 캐릭터입니다.

- 클래스: 마법사
- 등급: SR
- 획득: 시즌 패스 또는 5,000 다이아
- 주력 스킬: 파이어볼, 메테오
- 특수: 매주 화요일 던전에서 출현
```

### 5.2 사전 KB가 NER로도 쓰일 때
- 캐릭터·아이템명은 시스템의 명명된 엔티티 인식(NER) 사전으로도 활용.
- 같은 이름이 여러 의미일 때(예: "아리아"가 캐릭터명·아이템명·NPC명 모두) **disambiguation 표기** 필수.

## 6. Citation 친화적 구조

AI가 KB를 인용하려면 다음이 명확해야 함:

| 인용 요소 | 어디서 가져오나 |
|---|---|
| KB명 | `title` |
| 버전 | `version` |
| 섹션 | 본문 내 `#` 헤더 |

예: `[출처: 결제 환불 정책 v1.2 §환불 가능 조건]`

### 6.1 헤더 사용 가이드
- 본문에 `#`, `##` 헤더로 섹션 분리.
- 헤더가 그대로 citation §에 들어감.
- 헤더 텍스트는 짧고 명확하게.

### 6.2 인용 가능 단위 검사
- 한 KB 안에 헤더가 너무 많으면 분해 검토.
- 헤더 없이 한 단락만 있으면 OK (전체가 단위).

## 7. PII 회피 가이드

### 7.1 절대 본문에 쓰지 말 것
- 실제 유저 이메일·전화·이름 (가상 사례라도 X — 시스템이 검출).
- 실제 결제 ID·카드 마지막 4자리.
- 지갑 주소·tx hash (예시도 위험 — 가짜 주소 명시 권장).
- 직원·운영자 실명·연락처.

### 7.2 가짜 예시 표기
- "예: user@example.com" (`example.com` 도메인은 표준 안전).
- "예: tx 0x0000…0000" (모두 0인 명백 가짜).
- "예: 카드 마지막 4자리 ****" (별표).

### 7.3 시스템이 막는다고 방심 ❌
- PII 스캐너가 거부해도 작성자 권한 회수 가능 (TENANT_ONBOARDING §5.2).
- 작성 단계에서 의식적으로 회피.

## 8. 다국어 작성

### 8.1 작성 방식 옵션 (Q-013 결정 전)
- **A. 영어 원본 + 각 언어 직접 번역** — 정확도 ↑, 비용 ↑
- **B. 모국어 원본 + 자동 번역** — 비용 ↓, 정확도 ↓
- **C. 언어별 독립 작성** — 자유도 ↑, 일관성 ↓ (현재 권장 안 함)

### 8.2 권장 (잠정)
- **정책 문서**는 옵션 A 권장 (법적 정확도).
- **FAQ**는 옵션 B 허용 (검수 후 발행).
- **캐릭터·아이템 사전**은 게임 내 표기 우선 (게임이 다국어 미지원이면 영어).

### 8.3 작성 시
- `language` 메타 필수.
- 같은 토픽 다국어 KB는 같은 `slug` + 언어 suffix (예: `refund-policy.ko`, `refund-policy.en`).
- 한 언어 버전 변경 시 다른 언어도 동기화 책임 명시.

## 9. 검수 통과 팁

### 9.1 자주 걸리는 자동 검수 실패

| 실패 | 원인 | 해결 |
|---|---|---|
| PII 검출 | 본문에 실제 이메일·번호 포함 | 가짜 예시로 교체 (§7.2) |
| Citation 메타 불완전 | `version` 또는 `source` 누락 | 메타데이터 채움 |
| 언어 불일치 | 선언은 ko, 본문은 영어 | 본문 또는 메타 수정 |
| 모순 감지 | CROSS 표준 정책과 충돌 | cross_admin과 조율 |
| 중복 | 동일 슬러그 또는 본문 80%+ 일치 | 통합 또는 슬러그 변경 |
| 길이 초과 | 한 문서 N자 초과 | 분해 |

### 9.2 수동 승인 통과 팁
- 승인자가 5초 안에 핵심 파악 가능하게 구성.
- 첫 문장에 답이 있어야 함.
- 모호한 표현·법적 부담 표현 피함.

## 10. 흔한 실수 / Anti-patterns

### 10.1 ❌ 거대한 한 문서
- 환불·결제·계정 정책 한 KB에 다 넣음.
- → AI가 어느 부분을 인용해야 할지 모름.
- ✅ 토픽별 분해.

### 10.2 ❌ "참조 가이드를 보세요"
- 본문에 답이 없고 외부 링크만.
- → 유저에게 안 보임, AI도 인용 불가.
- ✅ 핵심을 본문에 포함, 보충 자료는 별도 KB 링크.

### 10.3 ❌ 내부 코드명·약어
- "RP-1234 발생 시" / "ITM-SLOT-3"
- → 유저는 이해 못함, 검색도 안 됨.
- ✅ 유저가 게임에서 보는 표기 그대로.

### 10.4 ❌ 시즌·이벤트 정보가 영구 KB로
- 이미 끝난 이벤트 KB가 검색에 잡힘.
- → 잘못된 정보 인용.
- ✅ `valid_until` 필수 + 종료 시 자동 비활성.

### 10.5 ❌ 추측·미확정 정책
- "환불 처리될 가능성이 있습니다" 같은 모호한 표현.
- → AI가 단정 답변하거나 유저 혼란.
- ✅ 확정된 정책만 KB로, 미확정 사례는 사람 라우팅.

### 10.6 ❌ "관리자에게 문의"
- KB 답이 "관리자에게 문의해주세요".
- → AI가 답할 수 없음, 자동 해결률 0.
- ✅ 가능한 한 KB에서 자체 해결 가능한 답.

### 10.7 ❌ 정책 표현 모호
- "보통", "일반적으로", "대부분의 경우"
- → 분쟁 시 책임 불명확.
- ✅ 조건 명시 ("다음 3가지 모두 충족 시").

## 11. 예시 모음

### 11.1 FAQ — 짧고 단순
```markdown
---
title: 비밀번호를 잊어버렸어요
slug: forgot-password
category: account
language: ko
version: v1.0
valid_from: 2026-05-19
tags: [password, account, recovery]
source: 운영 정책 §4.2
---

로그인 화면 [비밀번호 찾기]로 등록한 이메일에 재설정 링크를 보내드립니다.

이메일이 안 오시면:
- 스팸함 확인
- 가입 시 사용한 이메일이 맞는지 확인
- 30분 이상 지연 시 사람 상담사 연결
```

### 11.2 정책 — 조건과 예외
```markdown
---
title: 시즌 패스 환불 정책
slug: season-pass-refund
category: account
language: ko
version: v1.1
valid_from: 2026-05-01
tags: [refund, season, payment, policy]
source: 정책팀 결재 2026-04-28
---

# 환불 가능 조건

시즌 패스는 다음 조건 모두 만족 시 환불 가능합니다:
1. 구매 후 24시간 이내
2. 시즌 패스로 획득한 보상을 하나도 수령하지 않은 상태
3. 결제 수단에 정상 청구된 경우

# 환불 불가 케이스
- 보상 수령 후
- 24시간 경과
- 시즌 중 정책 변경 — 이전 정책 적용

# 환불 절차
1. 위젯 또는 이메일로 환불 요청
2. 결제팀 24시간 내 확인
3. 승인 시 영업일 3~5일 내 환급
```

### 11.3 캐릭터 사전 — NER 겸용
```markdown
---
title: 보린 (캐릭터)
slug: character-borin
category: faq
language: ko
version: v1.0
tags: [character, warrior, free]
source: 기획서 §2.1 캐릭터 도감
disambig: 길드명 "보린"과 다름
---

**보린**은 전사 클래스의 무료 캐릭터입니다.

- 클래스: 전사
- 등급: R
- 획득: 기본 제공 (튜토리얼 완료)
- 주력 스킬: 강타, 방패 가드
- 특수: 모든 던전에서 사용 가능
```
