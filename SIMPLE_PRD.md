# 게임 고객지원 시스템 — 한눈에 보는 기획서

**v0.2 초안 · 2026-05-19** · 상세 사양 → `PRD.md` · 결정 사항 → `DECISIONS.md`

---

## 한 줄 요약

**게임 안에서 유저가 문의하면 AI가 즉시 1차 응대**하고, 환불·계정 변경처럼 사람이 처리해야 할 일은 **각 담당자에게 전달**하는 시스템.

여러 게임사를 동시에 운영할 수 있는 구조(다중 게임사 운영).

---

## 왜 만드나

1. 사람 상담사 부담 줄이기
2. 24시간 다국어 자동 응답 제공
3. **블록체인·지갑 관련 문의 처리** — 기존 외부 솔루션(Zendesk·Intercom 등)이 못 다루는 영역
4. 게임별로 흩어진 문의 이력을 한 곳에서 관리

---

## 어떻게 다른가 (차별점)

일반 AI 상담은 보통 이렇게 작동합니다:
> 질문 → 매뉴얼 검색 → 답변

우리는 이렇게 합니다:
> 질문 + **유저 정보 (결제 이력·게임 진행도·지갑 상태)** → 답변

차별점 3가지:
- **유저 정보를 먼저 본다** — 누구의 어떤 상황인지 알고 답변
- **블록체인 거래 상태 직접 조회** — 트랜잭션 해시 보고 정상/실패 안내
- **여러 게임 통합 관리** — 한 유저가 여러 게임 쓰는 경우 통합 가시성

---

## 처리 흐름

```
[유저가 게임 안 채팅창에서 문의]
              ↓
   AI가 어떤 종류인지 분류
   (계정·결제 / 게임 진행 / 버그 / 신고 / 블록체인)
              ↓
   유저 정보 자동 조회 (CORE-001 통합 저장소)
              ↓
   지식베이스 검색 (게임별 + 공통 매뉴얼)
              ↓
   출처가 있어야 답변 (없으면 답변 거부)
              ↓
   ┌──즉시 답변      → 유저에게 안내
   ├──담당자 전달    → 큐에 쌓임, 담당자가 나중에 처리
   └──위험 사안     → 즉시 운영 관리자에게 알림
              (자해·법적 위협·괴롭힘 등)
```

---

## ⚠️ 비실시간임을 명시

AI 응답은 빠르지만 (10초 이내), **사람이 처리해야 하는 일은 모두 비실시간**입니다.

- 게임 안 채팅창이라도 환불·복구 요청 = 즉답 안 함
- "처리 대기 중입니다. 처리되는 대로 안내드리겠습니다"라고 안내
- 구체적인 처리 시간은 약속하지 않음

---

## 핵심 원칙 3가지

1. **AI는 조회만, 변경 작업은 절대 안 함** — 환불·계정 변경·아이템 지급 모두 사람이 처리
2. **출처가 있어야 답변함** — 지식베이스에 없는 내용은 추측해서 답하지 않음
3. **개인정보 가린 후 AI에 전달** — 이메일·전화·결제정보 자동 마스킹

---

## 처리 어려운 일은 담당자에게 전달

AI가 즉답 못 하는 경우 **해당 담당자**에게 큐로 전달:

| 문의 유형 | 담당 |
|---|---|
| 결제·환불 | 결제 담당팀 |
| 지갑 분실·트랜잭션 실패 | 인프라 담당팀 |
| 다른 유저 신고 (욕설·매크로 등) | 모더레이션 담당팀 |
| 버그 리포트 | 게임사 운영팀 → 개발팀 |
| 위 분류 안 되는 일반 | 게임사 운영팀 |

전달 후 유저에게는 "처리 대기 중" 안내. 구체 시간은 명시 안 함.

---

## 누가 쓰나

- **유저** — 게임 안 채팅창으로 문의
- **운영 관리자** — 출시 시점에 **이 한 사람이 모든 큐 처리**
- **각 담당팀** (결제·인프라·모더레이션·게임사 운영) — 향후 인력 확보 시 분배

---

## 채널 (출시 시점)

| 순위 | 채널 | 특징 |
|---|---|---|
| 1 | 게임 안 채팅창 (게임 내 위젯) | AI 즉시 응답, 사람 처리는 비실시간 |
| 2 | 디스코드 봇 | 동상 |
| 3 | 이메일 | 처음부터 비실시간 (첫 응답 30분 이내) |

---

## 향후 확장 (별도 계획)

| 항목 | 비고 |
|---|---|
| 자사 웹사이트 본체 | 게임 없이 직접 접근하는 독립 고객센터 |
| 모바일 앱 본체 | iOS·Android 독립 앱 |
| 카카오톡 | 채널 심사 별도 |

설계 시 향후 확장 고려해서 데이터 구조·신원·권한 짜기.

---

## 출시 시점 운영 모델

- **사람 상담사 없음** — AI만 작동
- **24시간 자동 응답**
- **모든 언어 지원** — Claude AI가 응답 가능한 언어 전부
- 큐는 쌓이고, **운영 관리자 한 명이 가용 시간에 처리**
- 위험 사안 (자해·법적 위협 등)은 즉시 알림 받음

향후 인력 채용되면 큐별로 담당팀 배치.

## 일정

**2026년 6월 내 출시** — 한 번에 완성. 단계 분할 없음.

---

## 목표 수치

**제품 목표**
- AI 자체 해결 비율: 50% 이상
- 만족도: 5점 만점에 4.0 이상
- 오답 보고: 2% 미만
- AI 응답 시간: 게임 채팅창·디스코드 10초 이내, 이메일 30분 이내

**절대 지켜야 할 것 (0건 보장)**
- AI가 환불·변경 작업 실행
- 개인정보 마스킹 누락
- 출처 없는 사실 주장
- 게임사간 정보 유출

---

## 이번 출시에 안 만드는 것

- AI가 환불·계정 변경 직접 실행 (향후 별도 명세)
- 음성·전화 (영구 제외)
- 자사 웹·모바일 본체·카카오톡 (별도 계획)
- AI 자체 학습 (지식 검색만 사용)

## 의존하는 시스템

| 시스템 | 용도 |
|---|---|
| **CORE-001 통합 유저 정보 저장소** | 유저 정보 통합 (필수) |
| 로그인 SDK | 게임 안 채팅창 + 신원 |
| Hub Builder 권한 | 운영자 접근 권한 |
| Sentinel | 어뷰징 탐지 + 제재 항소 |
| 메시징 모듈 | 다국어 템플릿 |
| Claude AI | 응답 생성 AI 모델 |
| 게임사 API | 유저·결제·로그 조회 (조회 전용) |
| Firebase 인증 | 유저 신원 검증 |

---

## 유저 식별 방식

- Firebase 가입 ID 또는 게임사 발급 UUID
- 게임사별로 분리해서 저장 (한 유저가 여러 게임 쓰면 게임별 별도)
- 미가입 유저는 익명 처리, 변경 작업 금지

---

## 개인정보 처리 요약

- AI에 보내기 전 개인정보 자동 마스킹
- 보관: 일반 18개월, 분쟁 사례 7년
- "AI 학습 사용 동의"는 별도 항목으로 받음
- GDPR(유럽 개인정보보호법) 기준 우선 적용. 한국·미국 법규 매트릭스 따로

---

## 가장 큰 위험 3가지

1. **AI가 잘못된 정책 안내** (예: 환불 약속) → 시스템 강제 + 조회 전용 + 출처 강제
2. **악의적 입력으로 개인정보 누설** → 마스킹 + AI 격리 + 출력 검사
3. **게임사간 정보 유출** → 모든 데이터 조회 시 게임사 ID 강제 필터
